# 摩耶重炮 语音站 · 开发日志

> 项目:摩耶重炮(マヤノトップガン)角色语音在线播放网页
> 作者:周炜业(GitHub @Zhoumornnig)
> 技术:纯静态单页(HTML + CSS + 原生 JS),无框架、无依赖,可直接部署到 GitHub Pages。

---

## 项目简介

把 215 条摩耶重炮的日文角色语音做成一个可以自由选择、播放的网页:
天空飞行主题动态背景、中日双语标题、分类筛选、搜索、随机/连播,以及可拖动、可收起、会自动定位的悬浮播放器。

素材版权归 Cygames《赛马娘 Pretty Derby》所有,仅供个人欣赏。

---

## 目录结构

```
摩耶重炮/
├── index.html              网页本体(动态背景 + 播放器,单文件)
├── data.js                 215 条语音数据(标题 / 日文 / 分类),供网页载入
├── manifest.json           同数据的标准 JSON(含每条日文原名,可对照 / 还原)
├── audio/                  215 个音频,已改成安全英文名 001.mp3 … 215.mp3
├── 使用说明与部署教程.md
├── 开发日志.md              本文件
└── _build/                 构建脚本与源数据
    ├── filelist.txt        原始日文文件名顺序(对照锚点)
    ├── translations.mjs    中文翻译 + 分类源
    └── build.mjs           由源数据生成 data.js / manifest.json
```

## 更新日志

### v1.0 — 2026-07-10 · 初始版本
- 整理桌面 `摩耶重炮` 文件夹里的 **215 个 mp3** 语音素材。
- 将全部 **215 条日文台词翻译成中文**,并按 11 类归类:
  问候 / 日常 / 训练 / 出走 / 胜利 / 失落 / 成长 / 剧情 / 活动 / 演出 / 系统。
- **音频安全化**:原日文文件名带 `☆ ♪ ～ ！` 等特殊符号,部署后易乱码 / 加载失败;
  统一改名为 `001.mp3 … 215.mp3`(采用「复制 → 逐个校验 → 再删原文件」流程,原名完整记录在 `manifest.json`)。
- 编写 `_build/build.mjs`:由 `filelist.txt` + `translations.mjs` 自动生成 `data.js` 与 `manifest.json`,顺序严格对齐防止错位。
- 网页 `index.html`:
  - **天空飞行动态背景**(贴合"飞行员 / 战斗机"人设):渐变天空、漂移白云、掠过的飞机尾迹、闪耀星光。
  - 中日双语卡片、🔍 中 / 日文搜索、分类筛选(带各类计数)。
  - 底部播放器:上一条 / 播放暂停 / 下一条、🎲 随机、自动连播、进度拖动;快捷键 空格、← / →。

### v1.1 — 2026-07-10 · 播放器可拖动 / 可收起
- 反馈:固定在底部的整条播放器会挡住画面。
- 将播放器改为**底部居中的圆角悬浮面板**(不再整条横贯,天然少挡)。
- 新增 **拖动手柄**:可把播放器拖到屏幕任意位置(自动限制在可视范围内,支持鼠标与触屏)。
- 新增 **收起按钮**:一键缩成仅「拖动 / 上一条 / 播放 / 下一条」的小条,再点展开。
- **位置与收起状态记忆**:存入 `localStorage`,下次打开保持原样。

### v1.2 — 2026-07-10 · 播放自动定位
- 播放某段语音时,画面**平滑滚动定位**到对应卡片,并伴随一次粉色高亮脉冲。
- 手动点击卡片:就近定位(已在视野内则不打扰);自动切歌:居中定位,便于跟读。
- 为卡片设置滚动安全距离,避开顶部筛选栏与底部悬浮播放器。

### v1.3 — 2026-07-10 · 部署上线
- 单独建立 GitHub 仓库并推送,开启 GitHub Pages,得到公开访问链接。

### v1.4 — 2026-07-10 · 新增动图画廊
- 整理 `项目/gif/` 里的 75 张动图(实为 `.webp`):批量重命名为 `gif001…`,按内容哈希**去重**(删掉 1 张重复),连续补号为 **001–074**。
- 将 74 张动图拷入网站 `gallery/`,`_build/build.mjs` 增加扫描 `gallery/` 生成图片列表。
- 网页新增顶部 **「🎧 语音 / 🖼 动图」切换标签**:
  - 动图画廊为响应式网格,图片 **懒加载**(`loading="lazy"`),滚动到才加载,避免一次拉取 69MB。
  - 点击任意动图弹出 **灯箱放大**,支持 ‹ › 上/下一张、✕ 关闭、键盘 ← → / Esc,点背景关闭。
  - 灯箱打开时暂不响应播放器的方向键,避免冲突。

### v1.5 — 2026-07-10 · 画廊性能优化(缩略图)
- 问题:74 张动图(各约 1MB,合计 68MB)同时解码播放,浏览器负担过重,部分图**加载不全 / 空白**。
- 用 `sharp` 提取每张动图的**静态首帧**,生成 420px 缩略图 `gallery/thumbs/gifNNN.webp`(单张约 24KB,合计仅 1.8MB)。
- 画廊网格改为加载**静态缩略图**(秒开、全部可见),点开**灯箱时才加载完整动图**播放。
- 卡片加 `content-visibility:auto`,离屏不渲染,进一步减负。
- 构建脚本 `build.mjs` 输出 `{full, thumb}` 结构;新增 `_build/thumbs.mjs` 生成缩略图(幂等)。
- 注:`_build/node_modules/`(sharp,仅构建用)已在 `.gitignore` 中排除,不进仓库。

### v1.6 — 2026-07-10 · 画廊按需播放动图
- 需求:画廊里的图要"动起来",但 74 张同时播放又会卡。
- 方案:用 `IntersectionObserver` 做**视口内播放**——动图卡片默认显示静态缩略图,滚动进入视野时切换为完整动图(播放),离开视野换回缩略图释放资源。
- 效果:看到的永远是动的,但同一时刻只播当前屏那几张,流畅不卡;不支持该 API 的老浏览器降级为静态缩略图。

### v1.7 — 2026-07-10 · 画廊改回全部播放
- 按需求回到"所有动图直接播放"的效果:画廊网格直接加载完整动图(`loading="lazy"` 进入视野即加载)。
- 保留 `content-visibility:auto`:仅屏幕外的卡片不渲染,屏幕内的全部持续播放——观感即"全部都在动",同时避免 74 张同屏解码导致卡顿/加载不全。
- 缩略图仍保留(灯箱及潜在回退用)。

### v1.8 — 2026-07-10 · 画廊改回"看到才动"
- 按需求切回 v1.6 的**视口内播放**方案:动图默认显示静态缩略图,滚动进入视野时才切换为完整动图播放,离开视野换回缩略图释放资源。
- 兼顾动效与流畅:看到的都在动,同屏只播当前几张,不卡、更省流量。

### v1.9 — 2026-07-10 · 画廊分页
- 画廊改为**分页展示**,默认每页 12 张;底部页码 + 上/下一页,页数多时自动折叠为「首页 … 当前 … 末页」。
- 翻页仅渲染当页动图(配合"看到才动"更省资源);灯箱翻页仍可浏览全部 74 张。
- 每页数量由 `index.html` 中的 `PAGE_SIZE` 控制。

## 如何重新生成数据

修改翻译或分类后(编辑 `_build/translations.mjs`),重新构建:

```bash
node _build/build.mjs   # 重新生成 data.js 与 manifest.json
```
