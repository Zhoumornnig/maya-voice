# 摩耶重炮 语音站 · 开发日志

> 项目:摩耶重炮(マヤノトップガン)角色语音在线播放网页
> 作者:周炜业(GitHub @Zhoumornnig)
> 技术:纯静态单页(HTML + CSS + 原生 JS),无框架、无依赖,可直接部署到 GitHub Pages。

---

## 项目简介

把 215 条摩耶重炮的日文角色语音做成一个可以自由选择、播放的网页:
天空飞行主题动态背景、中日双语标题、分类筛选、搜索、随机/连播,以及可拖动、可收起、会自动定位的悬浮播放器。

素材版权归 Cygames《赛马娘 Pretty Derby》所有,仅供个人欣赏。

---

## 目录结构

```
摩耶重炮/
├── index.html              网页本体(动态背景 + 播放器,单文件)
├── data.js                 215 条语音数据(标题 / 日文 / 分类),供网页载入
├── manifest.json           同数据的标准 JSON(含每条日文原名,可对照 / 还原)
├── audio/                  215 个音频,已改成安全英文名 001.mp3 … 215.mp3
├── 使用说明与部署教程.md
├── 开发日志.md              本文件
└── _build/                 构建脚本与源数据
    ├── filelist.txt        原始日文文件名顺序(对照锚点)
    ├── translations.mjs    中文翻译 + 分类源
    └── build.mjs           由源数据生成 data.js / manifest.json
```

### v1.5 — 2026-07-10 · 画廊性能优化(缩略图)
- 问题:74 张动图(各约 1MB,合计 68MB)同时解码播放,浏览器负担过重,部分图**加载不全 / 空白**。
- 用 `sharp` 提取每张动图的**静态首帧**,生成 420px 缩略图 `gallery/thumbs/gifNNN.webp`(单张约 24KB,合计仅 1.8MB)。

- 卡片加 `content-visibility:auto`,离屏不渲染,进一步减负。
- 构建脚本 `build.mjs` 输出 `{full, thumb}` 结构;新增 `_build/thumbs.mjs` 生成缩略图(幂等)。

### v1.6 — 2026-07-10 · 画廊按需播放动图
- 需求:画廊里的图要"动起来",但 74 张同时播放又会卡。
- 方案:用 `IntersectionObserver` 做**视口内播放**——动图卡片默认显示静态缩略图,滚动进入视野时切换为完整动图(播放),离开视野换回缩略图释放资源。
- 效果:看到的永远是动的,但同一时刻只播当前屏那几张,流畅不卡;不支持该 API 的老浏览器降级为静态缩略图。

### v1.7 — 2026-07-10 · 画廊改回全部播放

- 保留 `content-visibility:auto`:仅屏幕外的卡片不渲染,屏幕内的全部持续播放——观感即"全部都在动",同时避免 74 张同屏解码导致卡顿/加载不全。
- 缩略图仍保留(灯箱及潜在回退用)。

### v1.8 — 2026-07-10 · 画廊改回"看到才动"

### v2.5 — 2026-07-10 · 修复背景滚动时重复
- 问题:天空渐变原本画在随内容滚动的 `body` 上,长页面(画廊)滚动时渐变被拉伸/错位,显得重复不美观。
- 修复:把渐变移到固定在视口的 `.sky` 层(`position:fixed`),整个背景钉在屏幕上,滚动时不再重复;`body` 仅留纯色兜底。

### v2.6 — 2026-07-10 · 动图加载优化(压缩 + 按页加载)
- 原因:动图每张多达 160 帧、512×512,74 张合计 68MB,又要求全部预加载,一打开就下 68MB,故慢。
- 生成压缩版动图 `gallery/anim/`(256px,`_build/anim.mjs`):**68MB → 33MB**(省约 51%)。
- 画廊网格改用压缩动图播放,**点开灯箱仍用原图**(清晰度不受影响)。
- 去掉"全部预加载",改为**按页加载**:只加载当前页的动图(每页约 5MB),打开与翻页都快。
- 背景飞过的动图也改用压缩版。

### v2.7 — 2026-07-10 · 修复动图加载干扰音频播放
- 现象:音频能播,但动图下载抢占带宽/资源,导致播放卡顿、断续。
- 主修复:给所有动图 `<img>` 设 `fetchPriority='low'`(画廊图 + 背景飞过的动图),让浏览器优先保障音频、其次再加载动图,不再互抢。
- 顺带增强:音频播放失败时用 blob(强制 `audio/mpeg`)兜底,应对个别浏览器对 GitHub Pages 非标准 MIME(`audio/mp3`)的严格校验;其它错误显示在播放条便于排查。

### v2.8 — 2026-07-10 · 动图再压缩

- 每页 12 张约 4MB,加载更快、更不容易与音频抢带宽;点开灯箱仍用原图,清晰度不变。

### v2.9 — 2026-07-10 · 动图再压缩(第二档)
- 压缩动图从 200px/q38 降到 **176px/q32 + alphaQuality45 + smartSubsample(effort6)**:**24MB → 15MB**。
- gif001 184KB、gif057 660KB;每页约 2.5MB。灯箱仍用原图,放大清晰。
- 累计:原图 68MB → 画廊实际加载 15MB。

### v3.0 — 2026-07-10 · 动图减帧(体积大砍)
- 需求:动图帧数太多(每张约 160 帧)。无 ffmpeg/gifsicle,改用 sharp:整段正确解码 → **每 3 帧留 1**(隔帧抽取)→ `join.animated` 重组为新动图,并拉长每帧停留保持原速。
- 效果:**15MB → 5.7MB**;gif057 从 160 帧/659KB 降到 54 帧/244KB,gif001 68KB。帧率约 33fps → 11fps,角色循环动画仍顺。
- 累计:原图 68MB → 画廊实际加载 **5.7MB**(缩约 92%);每页 12 张不到 1MB。灯箱仍用原图,放大清晰。
- 参数在 `_build/anim.mjs`:`WIDTH` 尺寸、`KEEP` 抽帧比例。

## 如何重新生成数据

修改翻译或分类后(编辑 `_build/translations.mjs`),重新构建:

```bash
node _build/build.mjs   # 重新生成 data.js 与 manifest.json
```
